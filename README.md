# BookAI

Spring Boot приложение для автоматизации работы барбершопа с клиентами на базе AI.

## Возможности

- Общение с клиентами через AI-ассистента
- Ответы на вопросы об услугах, ценах и записи

## Технологии

- Java 25, Spring Boot 4.0.3
- Anthropic Claude API

## Фиксация статуса проекта

### Task 1–6: Базовая инфраструктура

| Задача | Описание |
|--------|----------|
| Task 1 | Инициализация Spring Boot проекта, базовая структура пакетов |
| Task 2 | Интеграция с Anthropic Claude API (`AnthropicClient`) |
| Task 3 | REST-эндпоинт `/api/book` — простой LLM-запрос и ответ |
| Task 4 | Параметр `temperature` для управления креативностью ответов |
| Task 5 | Выбор версии модели (Haiku / Sonnet / Opus) |
| Task 6 | Агент: базовая multi-turn сессия в памяти (`BookAgent`) |

### Task 7: Сохранение контекста агента

**Задача:** агент должен помнить историю диалога после перезапуска приложения.

### Анализ: насколько текущий агент соответствует каноническому определению

> Агент — это функция от 6 параметров. LLM — это класс или интерфейс, агент — конкретная реализация с суженным контекстом под задачу.
> — Алексей Гладков

| Параметр | Что должно быть | Статус |
|----------|-----------------|--------|
| **LLM / Модель** | Мозг агента | ✅ `AnthropicClient`, выбор модели через UI |
| **System prompt** | Личность, роль и ограничения агента | ❌ Передаётся `null` — модель не знает кто она |
| **History** | История текущего диалога | ✅ `AgentSessionStore` + H2, переживает рестарт |
| **Memory** | Долгосрочная память о пользователе между сессиями | ⚠️ Есть история, но нет отдельного слоя памяти |
| **Tools** | Инструменты: поиск, калькулятор, внешние API | ❌ Отсутствуют |
| **Input / Output** | Структурированный вход и выход | ✅ `AgentChatRequest` / `AgentChatResponse` |

**Вывод:** текущий агент — это продвинутый stateful чат-бот. Настоящий агент отличается наличием agentic loop: LLM сама решает, какой инструмент вызвать, получает результат и принимает следующее решение. Этого цикла пока нет.

**Что нужно для полноценного агента:**
- `System prompt` — задать роль: _«Ты помощник барбершопа, помогаешь с записью и вопросами об услугах»_
- `Tools` — например, проверка доступных слотов, запись клиента, получение прайса
- `Agentic loop` — цикл: вызов LLM → решение об инструменте → выполнение → снова LLM
